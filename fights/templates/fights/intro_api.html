Hello and welcome to MMAAPI, a REST API for Mixed Martial Arts data. If you aren't sure what that means, don't worry. An API is just a way for computer programs to communicate and share information. If you just want to see the data you can do so in your browser.

Why make this?

As a huge fan of both MMA and sports analytics I have often wondered about things like "How well do fighters age?" and "Does a high finish rate predict future success?". After looking and not finding a good dataset to work with I decided to make my own and put an API on it so others can use it too.

This is a work in progress and I am actively adding more information. The database is updated daily and currently has all {{ fighters }} fighters, {{ fights }} fights, and {{ events }} events in UFC history. Going forward I would like to add more organizations and other interesting information such as betting lines for each fight.

How does it work?

The data is accessed through a typical REST format. I'll explain what that means but if you are familiar with that structure you might want to head straight to the Swagger Docs which break down all of the API endpoints.

All of the data is accessed from www.mmaapi.com/api/ + whatever type of data you want. Make a GET request through code or your browser to www.mmaapi.com/api/fighter/ and you will get a json object with a list of fighters and their id numbers. Add an id number to the url and will get more detailed information about that specific fighter.

For example the id for Amanda Nunes is 486. www.mmaapi.com/fighter/486/ returns an object with her bio information, all of her won and lost fights, and some career statistics.

The same can be done for fights. www.mmaapi.com/fight/ returns a list of events and www.mmaapi.com/fight/2335/ returns the following object: 

                
                {{ fight_ex }}

This works for events and any other resource in the database. Check out the docs page for complete documentation.
